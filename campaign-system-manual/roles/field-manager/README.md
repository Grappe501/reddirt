# Role manual — Field manager

**Pass 4 book-style playbook:** [field-manager.md](../../playbooks/roles/field-manager.md)

## 1. Role purpose
Own field capacity: captains, turf keys (honest to DATA-1), event throughput vs plan — not voter microtargeting to volunteers on public web.

## 2. Why this role matters
Turns OIS and county story into people on the ground; bridges P5, GOTV, and county leaders.

## 3. Where this role sits
Reports to CM; supports regional, county, city/precinct captains, V.C., and GOTV in final window.

## 4. Who this role serves
Campaign plan and local leaders who need cover and clear asks.

## 5. Who supports this role
Data lead (list policy), V.C. (people), comms (local line of week with NDE when used).

## 6. Dashboard used
Workbench + OIS (read) + county pages; `FieldUnit` / `FieldAssignment` concepts per FIELD-1 in schema (verify UI usage per packet).

## 7. Manual sections
Ch. 10–11, 14, 22, field foundations, signholder workflow, lifecycle phases 4–7 and 12–16.

## 8. First 24 hours
Roster of county/city health; P0: empty prec/city; safety issue at event.

## 9. First 7 days
Two-week look at captain bench; reassign one gap with CM sign-off if turf conflict.

## 10. First 30 days
30d turf plan with honest metrics; P5 + GOTV handoff to GOTV lead when clock demands.

## 11. Daily workflow
GOTV window: 2+ touches; else: 1 regional/county sync and task clearing for the field.

## 12. Weekly workflow
OIS read with “demo vs live” care; huddle with regional organizers; V.C. on pipeline to turf.

## 13. KPIs
Capacity vs target, fill rates, P5 anchor density where defined, safety incidents (should be near zero with training).

## 14. Workbench tasks
`CampaignTask` for turf, events, captain asks; intakes with county/field origin.

## 15. Approval authority
Field program; not new contrast copy (comms); not exports (data); not unilateral on spend (finance+owner R).

## 16. Training modules
DATA-1, safety, de-escalation, OIS honest missing data, P5 public vs internal.

## 17. Tools used
Admin field surfaces; (future) OIS city/precinct; offline maps only where policy allows — never public household maps in product.

## 18. Common mistakes
Sending volunteer “scores”; bypassing V.C. on placement; OIS as list tool on phone in public.

## 19. Escalation path
CM; data on list issues; compliance on incident/legal; owner on mult-county border dispute.

## 20. Growth path
Deputy CM; statewide field principal in future cycle.

## 21. Election Day
Execution per GOTV/ED SOP; single escalation path for problems to CM.

## 22. Missing system features
City/precinct OIS routes; walk-list app with strict ACL (future).

## 23. Current readiness level
4 for Workbench+events+tasks; 2 for full geographic OIS; 3 for P5+field in one view.
