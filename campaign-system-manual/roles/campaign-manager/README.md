# Role manual — Campaign manager

**Pass 4 book-style playbook:** [campaign-manager.md](../../playbooks/roles/campaign-manager.md)

## 1. Role purpose
Day-to-day execution: priorities, triage, field/comms/finance handoffs — primary human owner of the Workbench queue culture.

## 2. Why this role matters
Without a CM, the Campaign Operating System becomes a collection of UIs; the CM is the only role expected to read unified open work daily.

## 3. Where this role sits
Below owner, above all department leads; peers with data lead on PII policy, with compliance on legal edges.

## 4. Who this role serves
The candidate, staff, and ultimately voters through coherent execution.

## 5. Who supports this role
Owner (authority), compliance (legal), V.C. and field (throughput), admin (operators).

## 6. Dashboard used
Primary: `/admin/workbench`. Secondary: `/admin/tasks`, comms, calendar, events as the week requires.

## 7. Manual sections
Ch. 7, 12, 16, 20–22; workflows: CM onboarding, task queue, lifecycle.

## 8. First 24 hours
Read unified open work; set triage SOP; P0: stuck intake, legal-risk comms, PII export request.

## 9. First 7 days
Weekly huddle map; R assignment for county/intake; verify GOTV and ED SOPs exist on paper.

## 10. First 30 days
30-day scoreboard: open intake age, program milestones, burn vs budget (with finance).

## 11. Daily workflow
Morning: open work; midday: one field touch; EOD: handoff note if 24/7 on-call is split.

## 12. Weekly workflow
All-hands or async memo; OIS “demo vs live” check with comms+data; owner sync on risk.

## 13. KPIs
Median intake age, task cycle time, crisis response time (internal), error/retraction rate on comms.

## 14. Workbench tasks
All sources in `open-work.ts`: intakes, email workflow, campaign tasks, actionable threads, festival pending as applicable.

## 15. Approval authority
Everything except owner-only: spend caps, high-risk comms, voter export (must be data+policy), new contrast claims (compliance/counsel path).

## 16. Training modules
Workbench SOP, PII handling, RAG/author tools as internal only (not volunteer Guided surface).

## 17. Tools used
Prisma-backed admin UIs; `ADMIN` session; no production DB operations from chat or screenshots of secrets.

## 18. Common mistakes
BCC everyone on PII; treating OIS demo tiles as VAN; approving sends without read.

## 19. Escalation path
Owner (money, break-glass, press); compliance/counsel (legal, crisis); data lead (file).

## 20. Growth path
Principal or general consultant for future races; not “owner” without charter.

## 21. Election Day
Single command SOP: comms one voice; GOTV/field on execution; you own escalations to owner.

## 22. Missing system features
RACI object in DB; formal SLA on WorkflowIntake; mobile CM alert if desired.

## 23. Current readiness level
5 for Workbench spine; 3 for strategic automation; 2 for full CM without burnout if solo (need staff).
