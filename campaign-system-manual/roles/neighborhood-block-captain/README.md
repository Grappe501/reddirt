# Role manual — Neighborhood / block captain

## 1. Role purpose
Own **sub-precinct** or **neighborhood** coverage when **precinct** ID is not yet a product (DATA-1 / OIS).

## 2. Why this role matters
Bridges “too big to precinct, too small to city” in dense areas and campuses.

## 3. Where this role sits
Between **P5** team and **city** captain; still **aggregate-first** in public OIS when visible.

## 4. Who this role serves
Neighbors and **city** / **county** goals.

## 5. Who supports this role
**City** captain, **field** manager, **V.C.** for roster hygiene.

## 6. Dashboard used
**Planned** city OIS; today: **tasks** + huddle docs; public **OIS** **aggregates** only.

## 7. Manual sections
Ch. 10–11, `PRECINCT_SIGN_HOLDER...` (adjacent), 22, 5.

## 8. First 24 hours
Define **turf** boundary on paper/CRM policy; one contact to **city** field.

## 9. First 7 days
List of **unfilled** **blocks**; one recruit ask.

## 10. First 30 days
30d micro-map of activities (doors/tables) within **safety** rules.

## 11. Daily workflow
**Light** touch — usually **2–3** high-value touches/week at scale.

## 12. Weekly workflow
**City** sync; update **task** list for visibility/sign crew.

## 13. KPIs
**Block** coverage (definition fixed), **safety** incident count, join rate.

## 14. Workbench tasks
`CampaignTask` for “block gap”, **intakes** for interest from forms.

## 15. Approval authority
**Local** **event** rules; **not** new **comms** without comms.

## 16. Training modules
**Safety**, **de-escalation**, **data** boundaries; **no** raw voter browse on public web.

## 17. Tools used
**Tasks**; `VolunteerAsk`; **relational** (staff) if assigned.

## 18. Common mistakes
Publishing **house-level** maps; confusing **OIS** demo tiles with local truth.

## 19. Escalation path
**City** captain → **county** → **field** mgr; **safety** to **compliance** if legal.

## 20. Growth path
**City** captain or **precinct** when route exists; sideways to **event** lead.

## 21. Election Day
Own **turf** **visibility** and **trouble** first response per **SOP**; escalate incidents.

## 22. Missing system features
`City` OIS and **neighborhood** entity; until then, **string** + **metadata** in tasks.

## 23. Current readiness level
**2** — mostly **SOP+tasks**; **OIS** support **0–1** for this layer in routes.
