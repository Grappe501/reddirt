# Workflow — County leader onboarding

## Purpose

Install a **county** point person with access to the right public surfaces, admin tools (if any), and **honest** data culture.

## Entry point

- CM/field assignment; `admin/counties` roster workbench

## User role

- **County leader**; **regional** sponsor

## System role

- Public `/counties/[slug]`; optional `admin/counties/[slug]` editor when granted

## Data captured

- `County` records; `CountyElectedOfficial`/leads; strategy notes in admin; **no** public dump of PII

## Dashboard updates

- County command; county intel workbench; OIS (when they use it for comms)

## Workbench queue

- County-tagged intakes, tasks, event suggestions for that county

## Approval path

- **CM/owner** for tool access; **compliance** for sensitive content on county pages

## KPI affected

- County pipeline, event density, local narrative adoption

## Training needed

- DATA-1, county **Field Intelligence** limits, P5 when local program launches

## Current implementation status

- **4** public county; **3–4** staff county tools; **2** for unified single county “single pane of glass”

## Missing pieces

- One merged **CountyIntelligenceViewModel** (per COUNTY plan); OIS v2 for all counties

## Next build steps

- County packet for second pilot after Pope sign-off; update manual
