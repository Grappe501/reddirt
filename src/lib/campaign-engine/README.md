# Campaign engine (foundation seam)

**FND-1+** use this folder for **non-invasive** **unified campaign engine** vocabulary. It does not **replace** `lib/*-workbench` or domain modules; it **names** rails, layers, and **(ROLE-1)** **positions** for future packets.

- **Foundation doc:** `docs/unified-campaign-engine-foundation.md`
- **Rails matrix:** `docs/shared-rails-matrix.md`
- **Position system (ROLE-1):** `docs/position-system-foundation.md`, `docs/workbench-job-definitions.md`, `docs/position-hierarchy-map.md`
- **Talent / training (TALENT-1):** `docs/talent-intelligence-foundation.md`, `docs/position-development-matrix.md`, `docs/talent-recommendation-flow.md`
- **AI agent brain (BRAIN-1):** `docs/ai-agent-brain-map.md`, `docs/ai-integration-matrix.md` — *runtime lives in* `src/lib/openai/`, `src/lib/assistant/`, `src/lib/comms/ai.ts`
- **Alignment + override + user-scoped context (ALIGN-1):** `docs/campaign-brain-alignment-foundation.md`, `docs/automation-override-and-impact-foundation.md`, `docs/user-scoped-ai-context-foundation.md` — *types only:* `alignment.ts` (layers, source kinds, version hint), `overrides.ts` (override kinds, impact class, event draft), `user-context.ts` (scoped layer ids, user-understanding facets); no persistence, no RBAC, no hidden automation.
- **Assignment + UWR-1 + WB-CORE-1 (position workbench / inbox read):** `docs/assignment-rail-foundation.md`, `docs/position-inbox-foundation.md`, `docs/unified-open-work-foundation.md`, `docs/unified-incoming-work-read-model.md`, `docs/position-workbench-foundation.md` — `assignment.ts`; `open-work.ts` (UWR-1 queries; `getOpenWorkForPosition` = re-export of `getInboxForPosition` from `position-inbox.ts`); `position-inbox.ts` (heuristic `getInboxForPosition` / summary / high-priority); `personalized-workbench.ts` (types only, placeholder guidance slots). **Read-only UI:** `UnifiedOpenWorkSection` on main workbench; `/admin/workbench/positions` and `/admin/workbench/positions/[positionId]`.
- **SEAT-1 (position seating, staffing metadata):** `docs/position-seating-foundation.md`, `docs/delegation-and-coverage-foundation.md` — Prisma `PositionSeat` / `PositionSeatStatus`; `seating.ts` (`listPositionSeats`, `getCoverageSummary`, `getPositionWorkbenchSeatContext`, `getSeatAssignmentContext`, …); `positions.ts` (`isValidPositionId`, `ALL_POSITION_IDS`). **Admin:** `/admin/workbench/seats` (assign/clear per row; no RBAC, no UWR-1 side effects). Position workbench page shows seat banner when present.
- **ASSIGN-2 + SKILL-1 (seat-aware read + agent skills / ingest):** `docs/seat-aware-assignment-foundation.md`, `docs/agent-skill-framework.md`, `docs/agent-knowledge-ingest-map.md` — `assignment.ts` (`SeatAssignmentContext`, `SeatInboxWorkAlignment`); `open-work.ts` (`getOpenWorkForSeat`, `getOpenWorkForSeatOccupant`, `getSeatInboxWorkAlignment`, re-export `getSeatAssignmentContext`); `skills.ts` (`AgentSkillDomain`, ingest tiers, **types/consts** only). **No** auto-assign, **no** `positionId` on UWR-1 sources.
- **FUND-1 (fundraising desk / donor research / contactability / fundraising ingest — blueprint + types only):** `docs/fundraising-desk-foundation.md`, `docs/contactability-and-calltime-precheck-foundation.md`, `docs/donor-research-and-enrichment-foundation.md`, `docs/fundraising-kpis-and-goals-foundation.md`, `docs/fundraising-agent-ingest-map.md` — `fundraising.ts` (`ContactabilityStatus`, `DonorResearchSignal`, `FundraisingKpiKey`, …); `AgentSkillDomain.FUNDRAISING_OPERATIONS` in `skills.ts`. **No** dialer, **no** mass SMS, **no** live OpenFEC/AR ingest in this packet.
- **COMP-1 (compliance governance rail + paperwork simplification + compliance ingest):** `docs/compliance-governance-foundation.md`, `docs/compliance-paperwork-simplification-foundation.md`, `docs/compliance-skill-framework.md`, `docs/compliance-agent-ingest-map.md` — `compliance.ts` (`ComplianceDomain`, `ComplianceSignalKind`, `PaperworkPrepStage`, `ComplianceReviewStatus`, `ComplianceKnowledgeTier`); `AgentSkillDomain.COMPLIANCE_GOVERNANCE` (umbrella). **No** compliance workbench UI, **no** filing automation, **no** bank/API integrations in this packet.
- **POLICY-1 + COMP-2 + BUDGET-1 (policy defaults, first compliance file intake, spend governance types):** `docs/campaign-policy-foundation.md`, `docs/compliance-document-ingest-foundation.md`, `docs/budget-and-spend-governance-foundation.md`, `docs/budget-agent-ingest-map.md`, `docs/federal-state-coordination-foundation.md` — `policy.ts` (`CAMPAIGN_POLICY_V1` v1 defaults; **not** legal conclusions); `compliance-documents.ts` (labels, `COM2_PACKET`); `budget.ts` (`CostBearingWireKind`, `BUDGET1_PACKET`); `skills.ts` adds `CAMPAIGN_POLICY` and `BUDGET_GOVERNANCE`. **DB:** `ComplianceDocument` (admin `…/compliance-documents`); RAG on uploads **not** in this packet. **No** bank/ad APIs, **no** full budget workbench, **no** legal rules engine.
- **BUDGET-2 (budget structure + planned vs actual, on FIN-1):** `docs/budget-structure-foundation.md` (canonical for BUDGET-2); `budget.ts` (`BUDGET2_PACKET`, `COST_BEARING_WIRE_OPTIONS`, `isCostBearingWireKindId`); `budget-queries.ts` (`listBudgetPlans`, `getBudgetPlanDetail`, `getBudgetActualsByWire`, `getBudgetVarianceByLine` — **CONFIRMED** ledger only, category→wire); `policy.ts` (`SpendApprovalTier`, `spendBudget` on `CAMPAIGN_POLICY_V1`). **DB:** `BudgetPlan`, `BudgetLine`, `BudgetPlanStatus` (migration `20260511120000_budget2_budget_plan_line`). **Admin:** `…/budgets` (create plan + lines, variance table). **Not** commitments, **not** forecasting, **not** filing.
- **FIN-1 (ledger foundation + submission seams, not a finance product):** `docs/financial-ledger-foundation.md`, `docs/submission-to-ledger-bridge.md` — Prisma `FinancialTransaction` + enums; `budget.ts` extends with `DEFAULT_LEDGER_CATEGORY_TO_WIRE` and `getBudgetWireForTransaction`; `financial-ingest.ts` (`FIN1_PACKET`, `isFinancialSubmission`, `extractDraftTransactionsFromSubmission`). Admin `…/financial-transactions` list; **FIN-2** adds create/confirm. **No** bank sync, **no** filing, **no** auto-ledger from all submissions, **no** amount parsing from free text in FIN-1.
- **FIN-2 (ledger use + confirm + contribution type):** `docs/financial-ledger-foundation.md` (§6) — `confirmedByUserId` / `confirmedAt`, `FinancialTransactionType.CONTRIBUTION`. **`financial-ledger.ts`** — `createFinancialTransaction`, `confirmFinancialTransaction` · `src/app/admin/financial-transaction-actions.ts` · admin `…/financial-transactions` (minimal create + confirm). BUDGET-2 `budget-queries.ts` **excludes** `CONTRIBUTION` from **spend** actuals. **Not** import, reconciliation, or banking.
- **FIELD-1 (field geography + position/seat link):** `docs/field-structure-foundation.md` — Prisma `FieldUnit` / `FieldAssignment` (optional `positionSeatId`); `field.ts` — `listFieldUnits`, `getFieldAssignments`, `getFieldLeadersByCounty` (read-only). **Not** GIS, districts, or public `County` auto-sync.
- **YOUTH-1 (youth pipeline + ingest guardrails, types only):** `docs/youth-pipeline-foundation.md`, `docs/youth-agent-ingest-map.md` — `youth.ts` (`YouthStage`, `YouthRoleType`, `YouthEngagementSignal`). **Not** LMS, youth routing, or child-directed automation.
- **DATA-1 + COMMS-UNIFY-1 + IDENTITY-1 (documentation, no new code in packet):** `docs/data-targeting-foundation.md` (county goals, `VoterRecord.precinct` string, no P/T/B in Prisma) · `docs/communications-unification-foundation.md` + `docs/message-workbench-analysis.md` (orchestration map: threads, Tier 2 broadcast, Comms workbench, social, E-1) · `docs/identity-and-voter-link-foundation.md` + `docs/volunteer-data-gap-analysis.md`. **Not** a unified `Message` table, **not** voter scoring.
- **DBMAP-1 + LAUNCH-1:** `docs/database-table-inventory.md` (105 models) · `docs/launch-reengagement-foundation.md` · `docs/launch-segmentation-and-response-foundation.md` · `launch.ts` (`DBMAP1_PACKET`, `LAUNCH1_PACKET`, `LaunchResponseIntent`, `ReengagementPhase` / `ReengagementStatus`, **read-only** `countLaunchAudienceByKind`, `listLaunchReadySupporters`) · `scripts/print-prisma-inventory.mjs`. **Not** send automation, **not** new schema for launch waves.
