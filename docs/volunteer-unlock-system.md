# Volunteer unlock system (GAME-1) (RedDirt)

**Packet GAME-1 (Part D).** Defines **what changes** as volunteers advance: **tools**, **capacity**, **training**, **leadership powers**, and **visibility**. Unlocks are **earned trust**, not paywalls for basic civic participation.

**Cross-ref:** [`volunteer-leveling-system.md`](./volunteer-leveling-system.md) · [`volunteer-xp-model.md`](./volunteer-xp-model.md) · [`position-system-foundation.md`](./position-system-foundation.md) · [`assignment-rail-foundation.md`](./assignment-rail-foundation.md) · [`relational-ai-assist-foundation.md`](./relational-ai-assist-foundation.md)

---

## 1. Unlock categories

| Category | Purpose |
|----------|---------|
| **Content** | Templates, scripts, talking points—curated by comms/organizing. |
| **Capacity** | How many contacts / outreach logs / assignments per period. |
| **Training** | Modules gated so **readiness** matches **responsibility**. |
| **Leadership** | POD leadership, assignment rights, mentoring. |
| **Visibility** | Data and rollups appropriate to **role + level + jurisdiction**. |

---

## 2. Unlock matrix (by progression)

| Unlock | Minimum level (typical) | Dependency |
|--------|-------------------------|------------|
| **Starter message templates & scripts** | New Volunteer | Comms approval |
| **Expanded templates / variants** | Organizer | REL-2 contact logging live |
| **Advanced scripts + tone ladders** | Network Builder | ALIGN-1 / voice compliance |
| **New tool: “message template pack” drops** | Organizer+ | Content pack published |
| **Expanded contact capacity** (logging limits) | Organizer → Network Builder | Abuse monitoring |
| **Advanced training modules** (e.g. difficult conversations) | Network Builder | Training authorship |
| **Ability to lead a POD** | Regional Leader **and** staff seat alignment | [`pod-system-foundation.md`](./pod-system-foundation.md); not XP alone |
| **Ability to assign work** | Team Leader **and** scoped position/seat | ASSIGN-1; optional `PositionSeat` |
| **Deeper data visibility — county insights** | County Contributor | Geography + privacy rules; FIELD-1 |
| **Leadership dashboards (future)** | Team Leader / Regional Leader | GAME-2+ read models; aggregate only |

---

## 3. Interaction with ROLE-1 and seats

- **Unlock eligibility** (VPE) says what someone **may** be trusted with in-product.
- **Seat occupancy** (SEAT-1) says who **officially** holds accountability.
- **Rule:** *Assignment power* and *POD leadership* **require** both **level unlock** and **human staffing** (or explicit staff override) so VPE never silently grants org authority.

---

## 4. Interaction with REL-1 AI assist

- **Script unlocks** still obey **human send** and **no impersonation** ([`relational-ai-assist-foundation.md`](./relational-ai-assist-foundation.md)).
- AI may **suggest** next templates from **unlocked** sets only—no surfacing restricted packs.

---

## 5. Revocation / pause

- **Compliance** or **data abuse** investigations may **pause** unlocks without deleting history.
- Appeals and **provenance** should mirror TALENT-1 / email workflow audit patterns.
